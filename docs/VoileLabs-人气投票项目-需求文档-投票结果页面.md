### 术语定义

| 序号 | 名称 | 描述 |
| --- | --- | --- |
| 1 | 投票系统 | 新版中文东方人物人气投票（非结果页面部分） |
| 2 | 主问卷 | 用户社群调研、收集社群基本信息的问卷 |
| 3 | 分问卷 | 主问卷的附属问卷 |
|  | 额外问卷 | 对于社群某一个角度，深入调研的问卷 |
| 5 | 问题组 | 用户填写问题的基本单位，具备“隐藏“、”显示问题组中的某个问题“两种状态 |
| 6 | 相关问题 （related） | 在同一个问卷中，选择了某个问题的某个选项之后，本问卷中额外需要用户回答的问题 |
| 7 | 互斥选项 （mutex） | 在同一个问卷中，选择了某个问题的某个选项之后，本问题/本问卷的某个其他问题会被清除勾选/隐去选择的选项 |
| 8 | 选项组 （Group) | 问卷的多选题中，互斥的若干个选项（如选择了Group1的选项后，Group0的所有选项会自动取消勾选） |
| 9 | 票 | 参与投票的基本载体，代表一个账号在某个部门的完整投票权利，一张票包含多个票位 |
| 10 | 有效票 | 对应部门下，成功提交的、符合投票规范的票 |
| 11 | 票位 | 系统中投票的基本组成单位，每个票位可以容纳一个投票对象（角色/音乐/组合），分为本命票位和普通票位 |
| 12 | 本命票位 | 在某些统计数据中，加权为2的票位（普通票位加权为1） |

### 本届投票结果

**需求描述**：结果页面中角色部门的投票结果主表格页面，展示角色人气投票的完整排名数据，

**目标路径**：结果页面-首页-角色部门-本届投票结果

**权限范围**：所有用户

**页面内容**：

页面应展示如下内容：
页面顶部展示以下概览统计数据：

- 角色数：投票范围内的角色的数量
- 有效票数：角色部门下，成功提交的、符合投票规范的有效票的数量
- 本命票数：角色部门下，成功提交的、符合投票规范的票中，填写了“本命票位”的票的数量
- 平均得票数：平均每位角色所得到的票数（角色数 ➗ 有效票数）
- 中位得票数：从小到大进行排序，中间位次的角色的票数（如果中间位次有两个则取二者平均值）

结果说明：
- 本页面为角色部门的票数排行简表。页面打开时已按票数排序，本命加权是指1本命票计算为2票
- 更高级的搜索功能可以点击右下角的图标展开高级搜索框
- 如果在移动端觉得表格过于拥挤可以尝试打开浏览器的“电脑UA（桌面端网站）”功能
- 往届结果页面中的“同投率”一栏与“投票比例”合并
- 点 + 号或者表格行本身可展开详情，点击详情链接可以查看更加详尽的角色投票数据
- 点击表头可进行其他类型的排序
- 关于表格中的参数，可以在这里查看
  - 此处的“这里”为超链接，跳转到
    https://github.com/PatchyVideo/Touhou-Vote/blob/dev/doc/%E7%BB%93%E6%9E%9C%E9%A1%B5%E9%9D%A2%20-%20%E5%8F%82%E6%95%B0%E8%AF%B4%E6%98%8E.md

表格包含以下18个数据列，其中前2列为固定列（始终可见），其余12列支持横向滚动：
- 名次：取决于当前排序的标准，默认排序标准为“票数”
  - 同一个票数为同一名次；
  - 同一名次的先后顺序以“本命数”为先后顺序，如果本命数同样相同，则以角色系统ID为先后顺序；
  - 当有并列名次时，占据虚位，下一个名次要顺位递推，如第1名有2个角色则下一个角色为第3名
- 角色名：投票范围内的角色的中文名，译名以THBWiki给出的译名为准，出现争议译名的情况下将其他译名以括号的形式标注
- 票数：投票位（普通票位&本命票位）中，含有该角色的有效票的数量
- 本命数：本命票位为该角色的有效票的数量
- 本命率：本命票位为该角色的有效票在所有投票位中含有该角色的有效票中所占的比例（本命数 ➗ 票数）
- 本命加权：本命票位计算加权后，所计算出的该角色的票数（票数 + 本命数）
- 票数占比：所有有效票中，投票位（普通票位&本命票位）中含有该角色的有效票占比（票数 ➗ 有效票数）
  - 使用“高级搜索”中的“从投票中筛选”对结果进行筛选时，该参数名称改为“同投率”
- 本命占比：本命票位为该角色的有效票在所有的本命票中的占比（本命数 ➗ 本命票数）
- 男性数：调查问卷里的第一题“您的性别是？”选择选项“男”，且投票位（普通票位&本命票位）含有该角色的有效票的数量
- 男性比例：投票位（普通票位&本命票位）中含有该角色的有效票中，调查问卷的第一题选择选项“男”的有效票占比
- 女性数：调查问卷里的第一题“您的性别是？”选择选项“女”，且投票位（普通票位&本命票位）含有该角色的有效票的数量
- 女性比例：投票位（普通票位&本命票位）中含有该角色的有效票中，调查问卷的第一题选择选项“女”的有效票占比

展开详情：点击行首的“+”号图标或表格行本身可展开折叠详情面板：
- 日文名：投票范围内的角色的日文名，以THBWiki给出的日文名为准
- 所属作品类型：角色所属作品的类型，仅展开行详情时显示
- 所属作品：角色第一次登场的作品的中文名，以THBWiki给出的译名为准
- 初登场时间：角色第一次登场的时间，格式为“年（四位数字）/月（两位数字）/日（两位数字）”
- 占总体男性比例：调查问卷的第一题选择选项“男”的有效票中，投票位（普通票位&本命票位）中含有该角色的有效票占比
- 占总体女性比例：调查问卷的第一题选择选项“女”的有效票中，投票位（普通票位&本命票位）中含有该角色的有效票占比
- 角色详情：提供指向角色投票详情页面的链接

其他内容：
- 表格排序：默认按票数降序排列，票数相同时按本命数降序排列。点击任意列表头可切换排序字段和排序方向（升序/降序）
- 高级搜索：详见高级搜索

### 上届对比结果

**需求描述**：结果页面中角色部门的往届结果对比页面

**目标路径**：结果页面-首页-角色部门-上届对比结果

**权限范围**：所有用户

**页面内容**：

页面应展示如下内容：
页面顶部展示以下概览统计数据：与角色部门"本届投票结果"的概览统计相同

结果说明：
- 本页面为角色部门的往届结果对比表格，将本届投票结果与前两届并排展示。页面打开时默认按票数降序排列
- 表格中的"第⑩届"、"第⑨届"列分别对应当前届次的前一届和前两届数据。若某一届次中该角色未参选或未获得票数，则对应数值显示为"-"
- 名次变化以颜色区分：名次上升（名次数值变小）以绿色字体显示，名次下降（名次数值变大）以红色字体显示，名次不变以黑色字体显示。颜色基于与第10届的名次对比计算
- 往届数据列不参与排序。当点击表头切换排序字段时，仅当前届次（第11届）的数据重新排列，往届数据跟随对应行移动
- 更高级的搜索功能可以点击右下角的图标展开高级搜索框
- 点 + 号或者表格行本身可展开详情，点击详情链接可以查看更加详尽的角色投票数据
- 点击表头可进行其他类型的排序（票数、本命数、本命率三列支持排序）

表格包含以下17个数据列，其中前名次为固定列（始终可见），其余16列支持横向滚动：
- 名次：当前届次（第11届）的排名，默认按票数降序排列。排名规则与本届投票结果页面相同
  - 第⑩届：第10届（上一届）的排名名次
  - 第⑨届：第9届（上上届）的排名名次
- 角色名：投票范围内的角色的中文名，译名以THBWiki给出的译名为准
- 票数：第11届含有该角色的有效票的数量。支持排序
  - 第⑩届：第10届该角色的票数
  - 第⑨届：第9届该角色的票数
- 本命数：第11届本命票位为该角色的有效票的数量。支持排序
  - 第⑩届：第10届该角色的本命数
  - 第⑨届：第9届该角色的本命数
- 本命率：第11届本命数占票数的比例。支持排序
  - 第⑩届：第10届该角色的本命率
  - 第⑨届：第9届该角色的本命率
- 票数占比：第11届含有该角色的有效票在总有效票中的占比
  - 第⑩届（票数占比）：第10届该角色的票数占比
  - 第⑨届（票数占比）：第9届该角色的票数占比
- 本命占比：第11届本命票位为该角色的有效票在总本命票中的占比

展开详情：与角色部门"本届投票结果"的展开详情列相同（日文名、所属作品类型、所属作品、初登场时间、角色详情）

其他内容：
- 表格排序：默认按票数降序排列。支持点击票数、本命数、本命率列表头切换排序字段和排序方向（升序/降序）。往届数据列不参与排序，仅当前届次数据按选定字段重排
- 高级搜索：详见高级搜索，仅支持“从投票结果中筛选”。搜索仅作用于当前届次数据，往届数据列不受搜索关键词影响

### 投票演进比对

**需求描述**：结果页面中角色部门的投票演进对比页面

**目标路径**：结果页面-首页-角色部门-投票演进比对

**权限范围**：所有用户

**页面内容**：

页面应展示如下内容：
页面顶部概览区域展示以下统计信息：
- 总有效票数：角色部门下所有有效票的总数
- 总本命票数：角色部门下所有本命票位票的总数
- 总参投角色数：本届投票中角色部门的参投角色总数

使用说明：
- 本页面为单独角色投票的演进对比页面
- 请先选择需要对比的角色，最多可以选择十名角色

角色选择与操作流程：
- 角色选择区域包含一个虚线边框的容器，用于展示已选中的角色标签。每个标签以"角色名+删除按钮"的形式显示，点击删除按钮可移除该角色。若未选择任何角色，容器内显示灰色提示文字"请选择角色"
- 下方提供角色下拉选择框，列出所有参投角色。已选中的角色不会出现在下拉列表中
- 添加按钮：位于下拉选择框右侧，点击后将当前下拉选中的角色添加到已选列表。按钮状态受两个条件限制：已选角色数少于10且当前无查询请求进行中。满足条件时按钮为蓝色可点击态，否则为灰色禁用态
- 提交图表申请请求按钮：位于选择区域右侧（移动端独占一行显示在下方）。在已选角色数不为0且无查询请求时显示为蓝色可点击态，否则灰色禁用态。点击后向服务器请求选定角色的投票演进数据

投票演进：
- 提交请求后才会显示
- 区域说明：
  - 该图表表示该角色随着投票进程的票数变化情况
  - 其中减少的票数是因为投票人修改自己的投票去掉该角色而导致
  - 投票时间范围（示例："2022-06-17 18:00:00 至 2022-07-05 00:00:00"）
- 总票数演进
  - 折线图
  - X轴为投票时间（以小时为单位），Y轴为票数
  - 每条折线代表一个已选角色的票数随时间的演进轨迹
  - 鼠标悬停在投票曲线上时显示当前时间点和各角色对应票数
  - 图例：展示各角色对应的折线颜色和角色名
- 总本命票数演进
  - 子标题"总本命票数演进"下方展示折线图，结构与总票数演进图相同，但展示的是角色对应的本命票数的演进轨迹
  - 鼠标悬停在投票曲线上时显示当前时间点和各角色对应票数
  - 图例：展示各角色对应的折线颜色和角色名，点击图例可切换角色折线的显示状态

### 角色投票详情

**需求描述**：单个角色的详细投票数据页面，展示该角色的基本投票数据、投票演进趋势以及投票群体的问卷填写情况

**目标路径**：结果页面-首页-角色部门-本届投票结果/上届对比结果-角色投票详情

**权限范围**：所有用户

**页面内容**：

页面应展示如下内容：

页面顶部概览区域：
- 角色头像：该角色的头像图片
- 角色名称：该角色的中文名
- 数据统计网格：
  - 票数：投票位（普通票位&本命票位）中，含有该角色的有效票的数量
  - 本命票数：本命票位为该角色的有效票的数量
  - 本命率：本命票位为该角色的有效票在所有投票位中含有该角色的有效票中所占的比例
  - 票数全局占比：所有有效票中，投票位中含有该角色的有效票占比
  - 本命全局占比：本命票位为该角色的有效票在所有的本命票中的占比
  - 投票理由：该角色收到的投票理由总数，若数量大于 0 则以链接形式展示，点击跳转至角色投票理由页面

## 搜索与筛选组件

**需求描述**：投票结果页面提供的高级搜索与筛选功能组件，允许用户通过关键词搜索、筛选范围选择、附加约束条件等方式过滤数据。该组件适用于角色部门、音乐部门、组合部门的"本届投票结果"和"上届对比结果"页面，以及调查问卷结果页面

**目标路径**：/

**权限范围**：所有用户

**页面内容**：

搜索栏效果如下图所示：
- 搜索支持图形用户界面模式和指令模式
- 图形用户界面模式通过点击搜索面板底部的"更多功能请切换到指令模式"切换到指令模式
- 指令模式通过点击"返回图形用户界面模式"切换回图形用户界面模式
- 图形用户界面模式支持以下参数：
  - 关键字：字符串类型，输入需要搜索的关键字。默认值为空字符串
  - 搜索范围（角色部门、调查问卷）：定义关键词搜索的目标字段范围，有“所属作品”、“角色名”、“日文名”三种，可多选，至少选择其中的一种
  - 搜索范围（音乐部门、调查问卷）：定义关键词搜索的目标字段范围，有"曲目名"、"日文名"、"所在专辑"三种，可多选，至少选择其中的一种
  - 搜索范围（组合部门、调查问卷）：定义关键词搜索的目标字段范围，有"角色名A"、"角色名B"、"角色名C"三种，可多选，至少选择其中的一种
  - 最大得票数：数字类型，过滤出票数不低于该值的角色/曲目/组合。默认值为 99999
  - 最小得票数：数字类型，过滤出票数不低于该值的角色/曲目/组合。默认值为 0
  - 从投票中筛选：可添加附加约束条件，通过交叉引用投票数据，仅显示满足约束条件的投票者所投出的结果
  - 图形用户界面模式下，该功能包含三个子筛选区域，各区域内部为 OR 关系，区域之间为 AND 关系：
    - 角色部门投了以下角色：
      - 角色下拉选择框：用户从下拉框中选择角色名，可选择是否勾选"本命"以限定该角色为本命票位
      - 添加按钮：点击后，所选角色以标签形式展示在上方的虚线框内
      - 标签内容为角色名（若勾选本命则前缀"【本命】"），末尾带有 X 删除标记，点击标签即可删除该约束
      - 同一区域内支持添加多个角色，多个角色之间为 OR 关系（即投票者在角色部门投了任一指定角色即视为满足该区域条件）
      - 若未添加任何角色，虚线框内显示"请选择角色"的占位提示文本
    - 音乐部门投了以下曲目：
      - 包含一个曲目下拉选择框、一个"本命"复选框和一个"添加"按钮。交互逻辑与角色部门相同
    - 调查问卷回答了以下问题：
      - 用户从第一个下拉框中选择问卷问题，选择后出现第二个下拉框用于选择该问题的具体答案
      - 点击"添加"后将约束以"问题内容+答案内容"的标签形式展示在上方的虚线框内，点击标签可删除
      - 支持添加多个问题答案约束
      - 若未添加任何问卷约束，虚线框内显示"请选择问题"的占位提示文本
- 指令模式搜索逻辑如下：
  - 查询指令：文本输入框，输入查询指令，默认值为空
  - 指令模式支持的查询语法如下：
    - 问卷条件：q<问题 ID> = <选项 ID>
    - 示例：q11011 = 1101101
      问题 ID 和选项 ID 参考调查问卷数据定义（给出问卷数据的 Github 链接）
    - 角色部门投票条件：
      角色数组（OR 匹配）：chars: ["角色名 1", "角色名 2"]
      示例：chars: ["东风谷早苗", "博丽灵梦"]
      【注意】数组里的结果是以或的形式查询的，即示例中查询的结果是角色部门中投了"东风谷早苗"或"博丽灵梦"的票中，各角色的投票结果
    - 本命角色：chars_first="角色名"
      示例：chars_first="东风谷早苗"
    - 音乐部门投票条件：
      曲目数组（OR 匹配）：musics: ["曲目名 1", "曲目名 2"]
      示例：musics: ["信仰是为了虚幻之人", "Native Faith"]
      【注意】数组里的结果是以或的形式查询的，即示例中查询的结果是音乐部门中投了"信仰是为了虚幻之人"或"Native Faith"的票中，各曲目的投票结果
    - 本命曲目：musics_first="曲目名"
      示例：musics_first="信仰是为了虚幻之人"
    - 逻辑运算符：
      - AND：取交集（同一规则也可使用 AND 关键字进行并列）
        示例：q11011 = 1101101 AND chars: ["东风谷早苗"] AND chars: ["博丽灵梦"]
        【注意】示例中查询的结果是[问题 id11011 回答的结果是 1101101，且同时在角色部门投了"东风谷早苗"和"博丽灵梦"]的票中，各角色的投票结果
      - OR：取并集
        示例：q11011 = 1101101 OR chars: ["东风谷早苗"]
      - 括号：使用"("或")"调整优先级
        示例：(q11011 = 1101101 AND chars: ["东风谷早苗"]) OR musics_first="信仰是为了虚幻之人"
- 重置按钮：位于搜索面板底部，点击"重置"按钮将清空当前模式下所有已填入的搜索条件，恢复至默认值
- 搜索按钮：位于搜索面板底部，"重置"按钮右侧。点击"搜索"按钮将通过 URL 查询参数应用当前所有搜索条件并关闭搜索面板
  - 搜索前会进行输入校验
  - 若最大得票数非整数或小于最小得票数，弹出提示"请检查最大得票数的填写是否正确！"
  - 若最小得票数为负数或非整数，弹出提示"请检查最小得票数的填写是否正确！"
  - 若搜索种类未选择任何一项，弹出提示"搜索种类不能为空！"
  - 校验通过后，将搜索参数编码至页面 URL 并刷新结果。搜索内容在页面刷新或分享链接时会被保留
